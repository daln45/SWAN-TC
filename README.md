# SWAN-TC

SWAN-TC is a hybrid deep learning framework for nearshore wave profile prediction, combining a Transformer-based spatial model (SWAN-T) with a CatBoost bias correction model (SWAN-C), with wave breaking constraints applied during inference.

## Architecture

```
SWAN-T  ──►  raw Hs/Tm profiles (64 depth points)
                          │
SWAN-C  ──►  bias (ΔHs, ΔTm) at each buoy/time
                          │
               correction + breaking limit (Hs ≤ 0.78·h)
                          │
SWAN-TC ──►  final corrected Hs/Tm profiles
```

- **SWAN-T**: Transformer encoder that takes bathymetry depth profiles and physical forcing variables (wave height, wind speed/direction, mean wave period/direction, tide, alpc) to predict the spatial Hs and Tm profiles across 64 cross-shore depth points.
- **SWAN-C**: CatBoost multi-output regressor that predicts the systematic bias (ΔHs, ΔTm) between SWAN-T predictions and observations.
- **SWAN-TC**: Full prediction pipeline applying SWAN-C bias correction to SWAN-T outputs, followed by a physical wave breaking constraint (Hs ≤ 0.78 × local depth).

## Repository Structure

```
SWAN-TC/
├── requirements.txt
├── SWAN-T/
│   └── train.py          # Transformer training (data efficiency experiments)
├── SWAN-C/
│   ├── train.py          # CatBoost bias correction training

    └── predict.py        # Combined SWAN-TC inference pipeline
```

## Getting Started

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

### 2. Train SWAN-T

Update the data paths inside `SWAN-T/train.py` (`DataEfficiencyConfig`) and run:

```bash
python SWAN-T/train.py
```

### 3. Train SWAN-C

Update the data paths inside `SWAN-C/train.py` (`Config`) and run:

```bash
python SWAN-C/train.py
```

### 4. Generate SWAN-C bias predictions

```bash
python SWAN-C/predict.py
```

### 5. Run SWAN-TC combined prediction

Update model/scaler paths in `SWAN-TC/predict.py` (`PredictionConfig`) and run:

```bash
python SWAN-TC/predict.py
```

Results are saved to `results_swan_tc/` as CSV files per year.

## Data Format

Input CSV files are expected to contain columns:

| File | Key columns |
|------|-------------|
| `swan_depth_YYYY.csv` | `id`, `time` (YYYYMMDDhh), `depth010`–`depth199` |
| `wind_wave_initial_data_YYYY.csv` | `id`, `time`, `swh`, `wind_speed`, `wind_direction`, `mwd`, `mwp`, `tide` |
| `bias_data_YYYY.csv` | `id`, `time`, `bias_hs`, `bias_tm` |

## Notes

- All model weights and scalers are serialized with `pickle` / `torch.save` and must be generated by the respective training scripts before running prediction.
- SWAN-C bias predictions must be generated before running the SWAN-TC pipeline.
